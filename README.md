# TEAMTESTHUB

TEAMTESTHUB is a lightweight web app for testing one prompt across multiple LLM models in parallel.

It is built for red teaming, but also for everyday prompt testing, prompt engineering, and model comparison.

## GitHub Description

Test one prompt across multiple LLMs in parallel for red teaming, prompt engineering, and model comparison.

## Who this is for

- **Security teams** validating model safety and jailbreak resistance.
- **Prompt engineers** iterating on wording and structure across model families.
- **Product and research teams** comparing quality, speed, and behavior before rollout.

## What it does

- Runs the same prompt on selected OpenRouter models at once.
- Streams results as they return, with latency and status.
- Lets you retry a single model without rerunning everything.
- Saves run history and prompt templates in your browser.
- Exports results to JSON, Markdown, and CSV.

## Privacy model

- API key is stored in `sessionStorage` (current browser session only).
- Theme, templates, and run metadata are stored in `localStorage`.
- The app is frontend-only and deploys as static files.

## Quick start

```bash
cd frontend
npm install
npm run dev
```

Production build:

```bash
cd frontend
npm run build
```

Deploy the contents of `frontend/dist`.

## Repository structure

```text

    public/                # Static files copied as-is into build output
    dist/                  # Production build output (generated by npm run build)
```

## What to publish

- For GitHub: push the full repository (`prompt-testing-platform`).
- For hosting (IONOS): upload only the contents of `frontend/dist`.

## Production note

By default, the app calls `/openrouter-api`.
In production, proxy `/openrouter-api` to `https://openrouter.ai/api/v1`, or set `VITE_OPENROUTER_API_BASE` to your API base URL.
